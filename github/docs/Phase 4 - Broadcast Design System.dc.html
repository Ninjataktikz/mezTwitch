Phase 4 · Broadcast Design System
+ Phase 8 Motion Addendum · July 2, 2026

the web system,
at 1920×1080, in motion.

No new identity decisions. This document translates locked tokens to broadcast surfaces and extends the motion law for live production.

01 — color on broadcast
  • Full-frame scenes (starting soon, BRB, ending, offline): Ink #0E0E0E ground, Chalk #FAFAF8 text — approved decision (July 2026): the broadcast lives on ink, sitting naturally beside dark gameplay. Rhythm comes from typography and the point, not surface inversion; chalk grounds are reserved for print and social.
  • In-game elements sit on gameplay, which is visually hot. They use Ink at 92% opacity with Chalk text — never translucent white, never blur ("glassmorphism" is a gradient in disguise).
  • Signal: one element per frame. On gameplay this is the point in the stat strip. On dark grounds use Signal/Dark #6D7CFF.
  • CS2 HUD color set toward neutral white/low saturation so the game's own UI joins the system rather than fighting it.

02 — broadcast type scale (at 1920×1080)
display 160/128
Full-frame scene headlines ("starting soon."). Helvetica 700, −2% tracking, lowercase.
stat 44–56 mono
Elo, K/D, countdowns. Tabular numerals, always. Numbers never animate by tweening — they cut.
label 22–24 mono
UPPERCASE, +14% tracking. Minimum broadcast size — nothing smaller ships to a 1080p frame.
body 28–32
Sentence copy on full-frame scenes only. Helvetica 400, 1.5 line-height.
03 — geometry & the 1080p grid
  • Module: 8px at 1080p (scale ×2 for 4K exports). Frame safe margin: 48px all sides for full-frame scenes; overlay elements sit 24px off the frame edge.
  • Radius 0, shadow none, border 1px hairline — unchanged. On gameplay, hairlines are Chalk at 40% so they define edges without glowing.
  • Webcam: 384×216 (16:9, exactly 20% frame width), bottom-left above the health bar zone, 1px hairline, nameplate strip beneath in mono. No frame decoration — the crop IS the frame.
  • Stat strip: 320×88, top-left, deliberately covering the radar's screen position on the viewer's side (snipe guard). Wordmark left, elo/K-D mono right, the point as separator.
04 — phase 8: motion addendum ("snap / hold / cut")

The web law — snap (120ms) and hold — gains one broadcast verb:

snap
120ms, cubic-bezier(0.2,0,0,1), opacity/color only. Alerts enter and exit with snap. Countdown digits snap-cut, never roll.
hold
Scenes are still. No ambient loops, no floating particles, no breathing gradients. The countdown and the blinking point are the only living things on a wait screen.
cut
New. Scene transitions are hard cuts through a 240ms full-frame ink wipe: ink slides in (120ms), the point blinks once at center, ink slides out (120ms). This is the stinger. Nothing else is sanctioned for transitions.
  • Alert grammar: ink strip snaps in at top-center of gameplay (never near HUD corners) · mono label + name · point blinks once · holds 3s · snaps out. Identical skeleton for follow/sub/raid; only the label changes. Sound: a single soft tick, −18dB relative to game.
  • Logo reveal: letters do not animate. The point cuts in, blinks once, wordmark cuts in beside it. 480ms total, then hold.
  • Kill/clutch graphics: none live. The game already renders kills. Clutch typography belongs to clips and social, where mez. isn't mid-round.
05 — do / don't
do
Empty chalk fields · one blue square · mono numbers · hairlines · hard cuts · silence
don't
Glow · gradients · particles · camera borders · tickers · rounded anything · a second accent · easing theatrics
