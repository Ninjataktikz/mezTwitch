Phase 6 · OBS Production Architecture
July 2, 2026

the game gets the frames.
obs gets what's left.

CS2 performance is the product. Every choice below spends the minimum on production so aim doesn't pay for it.

01 — scene list (exactly six)
01 · starting
Browser source (starting-soon page, live countdown) + mic muted by default.
02 · game
Game capture (CS2, capture-specific-window) + webcam + stat strip. The default scene 95% of the time.
03 · review
Window capture (demo playback) at 1216×684 on the review scene layout + webcam.
04 · brb
BRB page. Mic auto-muted (tie mute to scene switch via hotkey macro).
05 · ending
Ending page. Raid target chosen while this shows.
06 · tech
Chalk field + mono "one moment." For crashes. Exists so panic never shows on stream.

Naming: scenes numbered 01–06; sources prefixed by scene (02-cam, 02-strip, all-alerts). Shared sources (alerts, mic) live in a nested "all" scene included everywhere — one edit updates every scene.

02 — canvas & encoding
  • Canvas 1920×1080 · output 1920×1080 · 60fps. He plays at high refresh; the stream doesn't need to. Do not stream 4K — Twitch transcodes aren't guaranteed and CS2 loses frames for nothing.
  • Encoder: GPU hardware encoder (NVENC/AMF), never x264 on a gaming PC — x264 steals CPU threads CS2 wants. Bitrate 6000 kbps CBR, keyframe 2s.
  • If frames drop: first lower stream output to 1664×936 or 30fps — never lower the game.
  • Recording (for YouTube/clips): separate higher-quality local recording (CQP 18–20) to a second drive, same encoder session.
  • In CS2: uncap fps but set fps_max to a stable value; fullscreen exclusive; multicore rendering on.
03 — audio (four channels, fixed)
mic
Noise suppression (RNNoise) + compressor + limiter. Target −12dB peaks.
game
−6dB. Footsteps are content; music isn't played over queue.
alerts
−18dB relative to game. One tick sound. Never louder than a footstep.
voip
Teammate comms routed but duckable — one hotkey mutes VOIP from stream (privacy + TOS safety with strangers in queue).
04 — hotkeys (muscle memory, one hand)

F13–F18 (or numpad if no macro keys): scene 01/02/04/05 · mic mute · VOIP mute. Scene switches during rounds are banned — switch between rounds only; the stinger covers the cut.

05 — weight rules
  • Max two browser sources loaded at once; scene pages unload when inactive ("shutdown source when not visible" on).
  • No video-file loops on any scene — the system's stillness is also a performance budget.
  • Static overlay elements exported as PNG, not browser sources, wherever nothing moves.
  • Alert queue capped at one at a time; alerts pause during clutch rounds via a single "focus" hotkey if ever needed.
